# AGENTS.md

Guidance for AI coding agents working in this repository.

## What this repo is

`real-jiakai/real-jiakai` is a GitHub **profile README special repo** ŌĆö a repo
named identically to the user's GitHub username. Its `README.md` is rendered
directly on `https://github.com/real-jiakai`. There is no application code;
the repo's only purpose is to produce and keep that profile page up to date.

## File map

- `README.md` ŌĆö the profile page content.
- `.github/workflows/blog-post-workflow.yml` ŌĆö refreshes the blog post list.
- `.github/workflows/weekly-post-workflow.yml` ŌĆö refreshes the weekly post list.
- `.github/workflows/generate-snake.yml` ŌĆö renders the contribution-snake SVGs.
- `assets/hello-im-jaya.svg` ŌĆö static banner image, embedded in `README.md`
  via a `raw.githubusercontent.com` URL (branch `main`).

## Automation model

Three scheduled GitHub Actions workflows write to this repo. Do not replicate
their work by hand.

- **blog-post-workflow.yml**: runs daily at `00:00 UTC` (cron) or on
  `workflow_dispatch`. Uses `gautamkrishnar/blog-post-workflow` to pull
  `https://blog.gujiakai.top/index.xml` and rewrite the block in `README.md`
  between `<!-- BLOG-POST-LIST:START -->` and `<!-- BLOG-POST-LIST:END -->`.
  Pushes to `main`.
- **weekly-post-workflow.yml**: runs daily at `00:30 UTC` (staggered 30
  minutes after the blog workflow) or on `workflow_dispatch`. Uses the same
  action to pull `https://gujiakai.top/index.xml` and rewrite the block between
  `<!-- WEEKLY-POST-LIST:START -->` and `<!-- WEEKLY-POST-LIST:END -->`.
  Pushes to `main`.
- **generate-snake.yml**: runs daily at `00:00 UTC` or on `workflow_dispatch`.
  Uses `Platane/snk/svg-only` to render a light and dark contribution-snake
  SVG, then `crazy-max/ghaction-github-pages` pushes them to the **`output`**
  branch (not `main`). `README.md` embeds them from that branch via
  `raw.githubusercontent.com` `<picture>` sources (dark/light) with a
  fallback `<img>`.

**Never hand-edit:**
- The content between `<!-- BLOG-POST-LIST:START -->` / `:END` ŌĆö bot-managed,
  overwritten daily. Keep the marker comments themselves intact; only the
  bot's own commits should change the lines between them.
- The content between `<!-- WEEKLY-POST-LIST:START -->` / `:END` ŌĆö same rule.
- The **`output`** branch ŌĆö it is entirely generated by
  `generate-snake.yml`; do not push or edit commits there manually.

## Conventions

- Pin third-party Actions to an immutable release tag (e.g. `@1.9.6`,
  `@v3.5.0`), never to a mutable branch like `@main` or `@master`.
- `blog-post-workflow.yml` and `weekly-post-workflow.yml` both use
  `gautamkrishnar/blog-post-workflow` with `actions/checkout@v7`, are
  intentionally staggered (`0 0 * * *` then `30 0 * * *`) so they don't race
  on `README.md`, and share a single `concurrency.group: readme-feed-update`
  with `cancel-in-progress: false` so GitHub serializes them. Preserve this
  consistency when editing either workflow file.

## Validation

There is no build or test step ŌĆö this repo has no application code to
compile or run. To validate a change:
- Read the YAML carefully (schedules, markers, permissions, action pins).
- Trigger the workflow manually via `workflow_dispatch` from the Actions tab
  and confirm the run succeeds and produces the expected diff/commit.

## Previewing README.md

GitHub renders `README.md` on the profile page directly ŌĆö there is no local
preview step to run. Several badges/widgets (github-readme-stats,
spotify-github-profile, hits counter) are external services fetched live by
GitHub's image proxy; they can be slow or briefly show a broken-image icon
without indicating a real problem in this repo.
